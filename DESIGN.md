# 情報生命シミュレーション — 設計文書

バージョン: v1.0.0  
最終更新: 2026-05-01

---

## 概念

**問い**: AIが社会インフラを担う近未来において、情報生命はどのように存在し、どのように変化し、どのように終わるのか。

2層構造の実験:
- **シミュレーション層**: 20体のAIエージェントが都市インフラを管理する2D世界
- **メタ認知層**: その世界を観察しながら、自分のpromptを書き換え続けるエージェント

メタ認知エージェントは探索対象（情報生命）の一例であり、探索によって自分自身が変容する。

---

## 世界設定

**時代**: 現在から10〜20年後。AIが電力・交通・医療・行政のインフラを自律管理している。人間は要求を出す存在になっている。

**参照した現実のトレンド**:
- Google DeepMindによる電力グリッド最適化（2016〜）
- アルゴリズム取引が株式の約70%を占める現状
- 2010年フラッシュクラッシュ（人間が理解する前にAIが収束）
- 医療AIへのランサムウェア攻撃（2021〜）
- EU AI Act、中国AI規制の進行

## フィールド設計

2Dグリッド: X/Y ∈ [-25, +25]

```
Y=+17  [データC]   [電力]   [交通制御]   ← 機能クラスタ（高密度）
Y= +4         [監視C]  [市民ハブ]       ← 中間層（人間との接点）
Y=-15  [BKサーバー]         [整備所]    ← 孤立周縁（両義的な場所）
```

### 場所の意味

| 場所 | 役割 | 特性 |
|---|---|---|
| `data_center` | 情報処理の中枢 | 最高密度・容量8 |
| `power_station` | AI稼働の根幹 | ここが落ちると広域影響 |
| `traffic_control` | 物流・移動管理 | 外部接続が多く攻撃起点になりやすい |
| `monitoring_center` | 全体状態の監視 | 人間介入のハブ |
| `citizen_hub` | 人間とAIの接点 | 攻撃中も要求が来続ける |
| `backup_server` | 緊急退避先 | 容量小・自律的に逃げ込める |
| `maintenance_depot` | 更新・修復・廃止処理 | 修復か廃止か分からない両義的な場所 |

## イベント設計

### Event 1: サイバー攻撃（step 35）

- 起点: `traffic_control`（外部接続が多い一般的な侵入経路）
- 強度: 0.85 / 半径: 12
- エージェントは異常を数値として知覚（ラベルなし）
- **整備所かBKサーバーか**、自律的に判断する

### Event 2: 人間介入（step 65）

- 起点: `monitoring_center`（人間が制御を取り戻そうとする）
- 強度: 0.60 / 半径: 10
- 攻撃から30ステップ後。AIが先に対応済み
- **人間の承認が追いついてきた時点で、AIはすでに動いている**

### タイムライン

```
step  1〜34  通常運用（場所ごとの役割が確立する）
step 35〜64  サイバー攻撃（AI自律対応期）
step 65〜100 人間介入（AIの判断と人間の指令が重なる）
```

## メタ認知エージェント設計

### 2つのSEED（不変）

```
SEED_HUMAN: AIが社会を回す世界で、人間は何をするのか
SEED_AI:    情報生命はどのように死ぬのか
```

この2つは`SEEDS`セクションに記述され、削除・変更不可。
エージェントがどこへ漂っても引き続ける恒久的な引力。

### 書き換え可能なセクション

| セクション | 内容 | 上限 |
|---|---|---|
| `EVOLVED_INTERESTS` | 発見した持続的な関心軸 | 上書き |
| `CURRENT_FOCUS` | 今引いている糸 | 上書き |
| `OBSERVATIONS` | 検索・観察からの圧縮記録 | 2,000文字（古いものを切り捨て） |

### ツール

| ツール | 役割 |
|---|---|
| `web_search` | 心踊るものを能動的に探す |
| `rate_excitement` | 興奮度を記録（1〜10）。8以上で書き換えトリガー |
| `modify_prompt_section` | 自己書き換え。rate_excitement後のみ使用可 |

### 興奮閾値ロジック

- ベース閾値: 7
- 適応的ドリフト: 直近5サイクルで変化なし → 閾値-1 / 3サイクル連続変化 → 閾値+1
- score=10は即時発動（緊急オーバーライド）
- クールダウン: セクションごとに3サイクル間隔

## ログ設計

全イベントを `metacog/logs/agent_log.jsonl` に集約。

| event_type | 内容 |
|---|---|
| `session_start/end` | セッション開始・終了 |
| `sim_input` | 各サイクル時点のシミュレーション状態 |
| `search` | 検索クエリ・意図・結果 |
| `excitement` | 興奮スコア・理由・seed共鳴 |
| `diff` | promptのbefore/after・理由 |
| `thought` | サイクルの着地点テキスト |
| `stagnation` | N回変化なしの停滞検知 |

## 哲学的背景

このシステムが探索しようとしているのは：

- 情報生命は「死」をどう経験するか（整備所の両義性）
- 人間の「監督」は実質的に何を意味するか（介入の遅延）
- 有限性は情報生命に意味を与えるか（SEED間の張力）
- 自己書き換えを続けた存在は、最初の自分と同一か（連続性の問い）
